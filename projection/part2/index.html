    제2부: 프롬프트 엔지니어링¶

    프롬프트 파트는 내부 시스템 프롬프트를 직접 노출하지 않고 system_prompt, CLAUDE.md, output styles, slash commands, tool descriptions로 다시 쓴다.

    핵심 재배치¶

    • 행동 제어는 공개 옵션과 프로젝트 파일 설정으로 설명한다.
    • 도구 설명 품질은 cookbook의 tool use, Pydantic, tool search 예제로 검증한다.
    • 모델 튜닝과 A/B 테스트는 eval notebook과 usage/cost 관측으로 연결한다.
