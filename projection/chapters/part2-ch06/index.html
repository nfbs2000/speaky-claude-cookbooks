    6장: 프롬프트를 통한 동작 제어¶

    public 신뢰도: high

    공개 SDK 기준 재작성¶

    공개판에서는 행동 제어를 프롬프트 문체가 아니라 observable behavior로 평가한다. 도구 선호도, 질문/확인 빈도, 결과 형식, 실패 시 재시도 방식은 notebook 실행 로그와 결과물로 비교한다. chief-of-staff, tool choice, eval notebooks가 이 장의 근거다. 좋은 지침은 모델의 행동을 과하게 묶는 문장이 아니라, 도구 선택과 산출물 검증이 쉬워지는 실행 계약이다. 그래서 각 패턴은 "어떤 tool call과 result 변화가 보이는가"로 평가한다.

    Cookbook 근거¶

    • Behavior control with CLAUDE.md and output styles
    • Tool choice behavior
    • Building evals
    • Generate synthetic test cases

    공식 문서¶

    • Modifying system prompts
    • Get structured output from agents
    • Track cost and usage

    공개 경계¶

    • 행동 제어 효과는 샘플 한 번으로 단정하지 않는다.
    • 내부 모델 튜닝 규칙처럼 보이는 설명은 eval 설계로 바꾼다.

    실습 방향¶

    • 동일 task를 두 프롬프트로 실행하고 tool count, error, result quality를 표로 기록한다.
    • eval notebook의 기준을 prompt instruction 검증에 적용한다.
